🛡️

AES-256 Meta Credential Encryption Active

All long-lived permanent access tokens, secret keys, and campaign configurations are encrypted using AES/CBC/PKCS5Padding before serialization in PostgreSQL, keyed to tenant-specific workspace salts.

🔒

Webhook HMAC Validation Guard (X-Hub-Signature-256)

Incoming WhatsApp events and SDR chat requests are verified using a custom HMAC-SHA256 handshake. Any request lacking a valid Meta application signature is rejected with a 401 unauthorized code to prevent spoofing.

🔑

Tenant WebSocket Segregation

Live events and decisions dispatches are segregated via tenant-scoped STOMP channels (/topic/workspace/{workspaceId}/...) ensuring absolute cross-tenant isolation and preventing real-time data leaks.
